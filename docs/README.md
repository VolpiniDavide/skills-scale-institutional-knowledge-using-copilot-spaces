# OctoAcme Project Management Process Docs

Welcome to the OctoAcme Project Management documentation space. This README provides a summary of the core processes and quick links to each key document used by OctoAcme teams.

## Overview

OctoAcme manages projects with a **customer-first, iterative** approach. Key principles include:

- **Customer-first:** prioritize customer value and usability in every decision.
- **Iterative delivery:** deliver small, testable increments to reduce risk and gather feedback early.
- **Clear ownership:** each project has a named Project Manager (PM) and Product Lead responsible for delivery.
- **Data-informed decisions:** measure impact and iterate based on evidence.
- **Psychological safety:** encourage open feedback, learning, and continuous improvement.

## Process Lifecycle

Projects at OctoAcme follow a structured lifecycle:

1. **Initiation** — Define the problem, align stakeholders, and complete the project one-pager and decision gate.
2. **Planning** — Run the kickoff, build the backlog, define the Definition of Done, map dependencies, and create a release plan.
3. **Execution** — Operate team rhythm (standups, reviews, retros), maintain the project board, manage PRs and quality, and escalate blockers early.
4. **Release** — Complete pre-release requirements, follow the deployment checklist, prepare rollback procedures, and publish release notes.
5. **Retrospective & Continuous Improvement** — Capture learnings, define action items, track improvements, and feed them back into the next cycle.
